JacobOS

A 32-bit kernel in C and assembly — custom bootloader, preemptive scheduler, paging, a persistent filesystem, and user programs in ring 3. The machine below is real: the same image that boots in QEMU, running in an x86 emulator compiled to WebAssembly.

Files persist across visits: the whole machine state, disk included, is saved to your browser's IndexedDB when you leave (or press Save session) and restored next time. Reset disk wipes it for a clean boot.

The road to ring 3

What happens between power-on and the shell prompt — every stage hand-written, no libraries, no firmware help beyond the BIOS.

I Boot sector 512 bytes of assembly load the kernel over INT 13h, build a GDT, enter protected mode, and jump to C.
II Interrupts A 256-entry IDT with stubs for every CPU exception; the 8259 PIC remapped so hardware IRQs stop colliding with faults.
III Memory A bitmap physical frame allocator, paging switched on, and a first-fit kernel heap behind kmalloc / kfree.
IV Tasking Preemptive round-robin kernel threads driven by the PIT timer, with cooperative green threads layered on top.
V Storage An ATA driver feeding a hierarchical inode filesystem through a write-back block cache — files persist across reboots.
VI Ring 3 An ELF loader, a TSS for the privilege transition, and int 0x80 syscalls — user programs run unprivileged and ask the kernel for everything.
